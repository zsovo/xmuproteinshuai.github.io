OpenSWAH的安装

安装Debian软件包

如果您使用的是基于Debian的Linux，我们建议使用存档中我们提供的deb-package 。它最容易与“ gdebi”一起安装，如果存储库中有相关性，它们会自动解决相关性。 

sudo apt-get install gdebi
sudo gdebi /PATH/TO/OpenMS.deb

如果遇到不可用软件包的错误，则可以尝试以下步骤：
当Qt5（或其软件包之一，例如qt5xbase）丢失时，通常您的Debian太旧而无法在其官方存储库中获得足够的最新版本。我们建议使用与构建相同的软件包（确保适应Qt版本和您的Debian / Ubuntu版本，此处为Xenial）：

sudo add-apt-repository ppa:beineri/opt-qt59-xenial
sudo apt-get update

然后再次尝试安装。如果成功完成上述Qt安装后，某个版本的ICU及其libicu丢失了，建议您在https://pkgs.org/上找到丢失的版本，并使用gdebi进行安装。您可以安全地安装多个版本的ICU。 为了确保工具的功能，请按如下所示将OPENMS_DATA_PATH变量添加到您的环境中（执行工具时会提示您此错误）：

export OPENMS_DATA_PATH=/usr/share/OpenMS

如果您使用第三方存储库（通过上述解决方法）安装Qt5，请不要忘记像这样提供提供的环境文件（再次注意版本号）

source /opt/qt59/bin/qt59-env.sh

THIRDPARTY应用程序的可执行文件可以在以下位置找到：

/usr/share/OpenMS/THIRDPARTY

将文件夹内部添加到PATH中，以方便使用我们的适配器。

通过软件包管理器安装

当前，在这些发行版的相应软件包管理器中也提供了针对Fedora，OpenSUSE，Debian和Ubuntu 提供的OpenMS打包版本（它们可能需要在第三方“科学”存储库中进行查找）。对于其他GNU / Linux发行版或获取该库的最新版本，您需要构建自己的OpenMS。
注意: 这些软件包不是我们直接维护的，我们不能保证与您自行构建时的行为相同。此外，它们的可用性和版本可能会更改，并且支持可能会受到限制（由于不可预见或未经测试的行为）。我们也建议不要将它们与Debian软件包并行安装。

自行构建

从Github获取源代码，并参阅如何在GNU / Linux上构建OpenMS来了解如何构建OpenMS。
注意: 通过OpenMS中的适配器工具使用的某些第三方软件可能还需要安装JavaVM。

运行（Bio）Docker映像

我们还不断更新对Docker的支持。可以通过https://hub.docker.com/u/openms/和 http://biocontainers.pro/生物容器存储库获取图像。在您要执行的操作上，可以通过以下命令之一拉取图像 

docker pull biocontainers/openms
docker pull openms/library
docker pull openms/executables
docker pull openms/pyopenms

成功安装docker之后（请参阅Docker文档）。
可以在我们的Github上找到更多用于构建不同类型映像的dockerfile（例如，在ArchLinux上对应于构建指令）：https : //github.com/OpenMS/dockerfiles
